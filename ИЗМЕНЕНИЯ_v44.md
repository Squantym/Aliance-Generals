# ИЗМЕНЕНИЯ v44 — окно «События», профиль v2, окна достижений

## 1. Окно «События» при заходе в игру (сводка «пока вас не было»)

Если игрока атаковали или объявили на него санкции, ПОКА ОН БЫЛ ОФФЛАЙН
(нет активности > 5 минут — тот же порог, что и везде), события копятся
и показываются одним окном при первом заходе:

- Атак на вас: N / Отбито: N / Поражений: N
- Потеряно денег: $X (суммарно)
- Потери: сетка картинок техники с ×N (клик — карточка юнита)
- Санкции, объявленные на вас: флаг + имя заказчика (xN) + сумма

Новые файлы/правки:
- `src/services/warReport.ts` — НОВЫЙ сервис-копилка (onAttack, onSanction,
  view, ack). Данные в `user.offlineWar`.
- `src/services/battle.ts` — хуки в обеих ветках (поражение защитника с
  деньгами/потерями и отбитая атака).
- `src/services/sanctions.ts` — хук в `declare` (агрегация по заказчику).
- `/api/me` → `pendingWarReport`; `POST /api/war-report/ack` — очистка
  после закрытия окна.
- Фронт: `App._showWarReport()` в app.js, стили `.events-popup` в style.css.
  Показывается первым — раньше подарков и окон достижений.

## 2. Профиль v2 (по макету)

- Шапка: имя (зелёное, подчёркнутое) + активный титул + онлайн-статус;
  строка «Ур. X · Ал. Y · Рейтинг Z»; статус + «редактировать статус».
- Крупный аватар слева (148×196), справа: «Звание» и блок «Статистика»:
  победы (свои+оборона), поражения (свои+оборона), убийства (фаталити),
  смерти (гибель при подрыве на мине — счётчик «Смертника»; если нужно
  считать иначе — правится в одном месте: `deathsCount` в player.ts).
- Баннер «Ощутите все преимущества и удобства игры!» + кнопка «Стать VIP»
  (ЗАГЛУШКА: неактивна) + «Нажмите, чтобы увидеть подробности» (тост).
- Блок «Боевая эффективность»: Атака / Защита / Шанс крита / Шанс уворота
  (только в своём профиле). Дубли этих строк из нижней карточки убраны.
- Бэкенд: в `publicProfile` добавлено поле `deathsCount`.
- Все прежние id обработчиков сохранены (edit-status, pf-avatar-btn,
  pf-attack, pf-spy, pf-msg, pf-sanction, pf-stats-toggle, ...).

## 3. Окна новых достижений (очередь)

- При взятии этапа достижение попадает в `user.pendingAchievements`
  (не больше 30). Работает и для достижений, полученных оффлайн
  («Битый», «Враг народа») — окна покажутся при первом заходе.
- Окно: «Вы получили новое достижение:» + название + описание с порогом
  и наградой + круглая картинка `/img/achievements/{achId}_{stage}.webp`
  + ссылка «отобразить» («(можно сменить в профиле)») + «Закрыть» и ✕.
- «Отобразить» ставит разблокированный титул активным (`/api/titles/set`).
- Несколько достижений за одно действие — окна по очереди: закрыл одно,
  открылось следующее, пока игрок не увидит все.
- `/api/me` → `pendingAchievements`; `POST /api/achievements/ack {id}`.
- Порядок попапов при входе: «События» → достижения → подарки админа.
